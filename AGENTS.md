# AGENTS.md

This file provides guidance to AI agents and contributors working on this Capacitor plugin.

## Quick Start

```bash
bun install
bun run build
bun run verify
bun run fmt
bun run lint
```

## Development Workflow

1. Install dependencies with `bun install`; never use npm.
2. Build with `bun run build` to compile TypeScript, run docgen, and bundle output.
3. Verify with `bun run verify` before submitting work.
4. Format with `bun run fmt` and lint with `bun run lint`.

### Individual Platform Verification

```bash
bun run verify:ios
bun run verify:android
bun run verify:web
```

## Project Structure

- `src/definitions.ts` - TypeScript API and generated docs source
- `src/index.ts` - Plugin registration
- `src/web.ts` - Web unsupported stub
- `ios/Sources/` - iOS unsupported stub for package compatibility
- `android/src/main/` - Android implementation
- `Package.swift` - SwiftPM definition
- `*.podspec` - CocoaPods spec

## API Documentation

API docs in the README are auto-generated from JSDoc in `src/definitions.ts`. Do not edit the `<docgen-index>` or `<docgen-api>` sections directly. Update `src/definitions.ts` and run `bun run docgen`.

## Versioning

The plugin major version follows the Capacitor major version. This package starts at v8 for Capacitor 8 compatibility.

## Common Pitfalls

- Use Bun for every package command. Use `bunx` for package binaries.
- Android builds target Java 21.
- `dist/` is generated by the build and should not be edited manually.

## Timeout Policy

- Keep CI, script, and runtime timeouts at 10 minutes or less. Use `timeout-minutes: 10` or lower in GitHub Actions and cap timeout values at `600000` ms, `600` seconds, or `10m` unless explicitly requested.
